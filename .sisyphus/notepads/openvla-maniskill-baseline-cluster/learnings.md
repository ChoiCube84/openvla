
- Task 1 baseline uses a small importable constants module at `experiments/robot/maniskill/defaults.py`, matching repo preference for lightweight Python defaults over new config frameworks.
- Contract values are machine-readable primitives (lists/dicts/ints/strings) so downstream launcher/runner tasks can import directly without parsing.
- Artifact root follows existing repository pattern under `rollouts/`, with ManiSkill-specific root set to `rollouts/maniskill`.
- `check_setup.py` now enforces deterministic machine-readable failure tags (`CHECKPOINT_MISSING`, `GPU_POLICY_UNAVAILABLE`, `MANISKILL_DEPENDENCY_MISSING`, `RENDERER_DEPENDENCY_MISSING`, `DISK_BUDGET_UNAVAILABLE`) and prints `SETUP_OK` on success.
- `estimate_runtime.py` uses a safe probe path (module import timing plus optional one-step `gymnasium.make(..., render_mode="rgb_array")` probe) and writes required estimate fields to `rollouts/maniskill/runtime_estimate.json`.
- `experiments/robot/maniskill/artifacts.py` now keeps artifact helpers independent from runtime/env logic: run-layout creation, JSON/JSONL writers, episode metadata assembly, and exemplar selection are pure utility functions.
- `create_run_layout(run_id)` creates `rollouts/maniskill/<run_id>/` with canonical `frames/`, `videos/`, `manifest.json`, and `episodes.jsonl` paths so downstream runner/rebake tasks can rely on stable layout.
- Exemplar selection is deterministic by iteration order: `select_exemplars()` returns first 2 successes and first 2 failures only, and `build_exemplar_manifest(..., task_ids=...)` emits explicit absent-class counters (`absent_success_count`, `absent_failure_count`).
- `write_summary()` enforces required summary schema keys from `SUMMARY_SCHEMA_KEYS` and always injects canonical artifact paths for machine-readable downstream consumption.
- Task 4 runner follows LIBERO-style control flow with draccus config + outer task loop + per-episode rollout while keeping ManiSkill scope fixed to defaults `TASK_IDS` only.
- ManiSkill observation handling is unstable across wrappers, so `extract_image_observation()` now probes explicit preferred paths first and fails fast with `OBS_SCHEMA_MISMATCH:` when no RGB source is valid.
- Action adaptation is safest when strict on shape and permissive on bounds: `adapt_action_for_maniskill()` rejects non-finite or wrong-shape actions (`ACTION_SHAPE_MISMATCH:`) and clips to action-space bounds when available.
- Episode terminal semantics are now explicit via `interpret_step_outcome()` so `terminated`/`truncated` do not get conflated with task success; success is read from `info` keys only.
- `experiments/robot/maniskill/rebake_videos.py` can now regenerate MP4s directly from `episodes.jsonl` + `manifest.json` metadata and saved frame directories, so rebake does not require loading model/processor or re-running env rollouts.
- Default rebake selection prioritizes `manifest.json` exemplar records (`exemplars.per_task.{task}.{success|failure}`) and deterministically falls back to ordered `episodes.jsonl` records when exemplar metadata is absent.
- A shell launcher can remain zero-argument for users while still supporting deterministic QA failure injection by using a presence-based env override (`OPENVLA_MANISKILL_VISIBLE_DEVICES_OVERRIDE`) instead of argument parsing.
- Parsing `summary_path=` and `average_success_rate=` from the full-run stdout log lets the launcher print final headline outputs (`run_dir`, `summary_path`, `average_success_rate`) without modifying the Python benchmark runner contract.
